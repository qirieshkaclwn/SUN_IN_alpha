# Схема доверия и E2E для клиентов

## Роли

1. **Сервер (CA/регистратор доверия)**  
   Подписывает клиентские сертификаты и публикует доверенный корневой сертификат `private_ca.crt`.

2. **Клиент**  
   Сам генерирует ключевую пару локально, создает CSR и отправляет на сервер для подписи.

3. **Получатель сообщения (другой клиент)**  
   Предоставляет свой публичный ключ (через свой сертификат), которым отправитель шифрует сообщение.

## Что проверяется в коде сейчас

0. Если у клиента нет `*.crt/*.key`, он сам генерирует RSA-ключ, делает CSR и отправляет `cert_enroll` на сервер.
1. Сервер подписывает CSR своим `private_ca.key` и возвращает `cert_enroll_response` с `client_cert`.
2. Клиент сохраняет выданные файлы как `<nickname>.crt` и `<nickname>.key`.
3. Клиент отправляет `auth_init` с `nickname` и `client_cert`.
4. Сервер проверяет сертификат:
   - подпись от `private_ca.crt`,
   - срок действия,
   - соответствие `CN == nickname`.
5. Сервер отправляет `auth_challenge` с nonce.
6. Клиент подписывает nonce приватным ключом и отправляет `auth_proof`.
7. Сервер проверяет подпись публичным ключом из `client_cert`.
8. Только после этого отправляется `auth_success`.

## Файлы на клиенте

- `client/private_ca.crt` — доверенный CA.
- `client/<nickname>.crt` + `client/<nickname>.key` — сертификат/ключ для конкретного никнейма.
- Если `<nickname>.crt/.key` не найдены, используется fallback: `client/client.crt` + `client/client.key`.

Без этих файлов авторизация будет отклонена.

## оикуды доверие
- В каждый клиент вшивается (pinning) публичный ключ/сертификат вашего CA.
- Клиент принимает только те клиентские сертификаты, которые подписаны этим CA.
- Если сертификат не подписан нужным CA — он отклоняется.

## Как это работает по шагам

1. Клиент A генерирует `clientA.key` и CSR.  
2. Сервер (CA) подписывает CSR и выдает `clientA.crt`.  
3. Клиент B делает то же самое и получает `clientB.crt`.  
4. Клиент A получает сертификат B, проверяет подпись CA.  
5. Клиент A шифрует сообщение на публичный ключ B.  
6. Сервер пересылает только шифротекст в поле `text`.  
7. Клиент B расшифровывает своим приватным ключом.
